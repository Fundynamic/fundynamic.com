    met hoge kwaliteit software

    Ik vind het bouwen van software het mooiste vak dat er bestaat.

    De wereld verandert continu en software moet met deze veranderingen om kunnen gaan. Dit doe je door verandering te omarmen en dit zowel in proces als techniek terug te laten komen. Met behulp van XP Practices, Scrum en Test Driven Development is dit mogelijk.

    Met bovenstaande practices heb ik veel ervaring en ik ben groot voorstander van deze manier van werken.

    Daarnaast heb ik een aantal projecten gedaan die op verschillende manieren zijn uitgevoerd. Van Prince2 tot echt Scrum. Ik geloof dat een Agile manier van werken de manier is om IT projecten te doen.

    Mijn ervaring ligt vooral in het bouwen van middel-grote websites. Hierin zit veel integratie met andere systemen en zijn er schaalbaarheid en performance uitdagingen.

    liefde voor het vak

    Always check a module in cleaner than when you checked it out.

    Liefde voor het vak uit ik op verschillende manieren:

  • het goed structureren van code door gebruik van design patterns
  • het geautomatiseerd testen van code door unit, controller, integratie tests
  • DRY, YAGNI en SOLID principes.
  • De boyscout rule. Laat het kamp (code) beter achter dan hoe je het aantrof.

    • Code wordt meer gelezen door mensen dan door computers. De kunst is het dan ook om code dusdanig op te zetten dat het begrijpelijk is voor mensen.

    Ik beschouw softwareontwikkeling als een creatief vak waarin je, net als een schilderij, van een grof naar steeds concreter en gedetaileerder beeld toe werkt.

    Als code met liefde wordt opgezet onstaat er een geheel wat goed samenwerkt.
